# 要件定義チェックリスト

## 1. プロジェクト基本情報

### 目的・ゴール

- [ ] プロジェクトの最終目標は何か
  - 例：「AI を活用した LP 生成により、制作時間を従来比 70%削減する」
- [ ] 具体的な成功指標（KPI）は何か
  - 例：「月間ユーザー数 1000 人」「LP 生成数 500 件/月」
- [ ] プロジェクトの優先順位（他プロジェクトとの関係）
  - 例：「Q2 の最重要プロジェクト」「マーケティング部門の基幹システム」
- [ ] 予算範囲
  - 例：「初期開発 500 万円」「月間運用コスト 30 万円以内」
- [ ] スケジュール制約
  - 例：「3 ヶ月以内のリリースが必須」「年末商戦に間に合わせる」

### ステークホルダー

- [ ] 主要なステークホルダーは誰か
  - 例：「マーケティング部門」「営業部門」「経営企画室」
- [ ] 意思決定者は誰か
  - 例：「プロダクトオーナー：山田氏」「最終承認者：佐藤部長」
- [ ] 実際のエンドユーザーは誰か
  - 例：「マーケティング担当者」「Web 制作者」「営業担当者」
- [ ] 運用・保守担当者は誰か
  - 例：「社内 IT 部門」「外部ベンダー」
